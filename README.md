# ai_company

> **AI 에이전트로 구성된 회사**  
> Spec-Driven · Agent-Oriented AI Organization Prototype

---

## 1. 프로젝트 개요

**ai_company**는  
AI를 단순한 도구가 아닌 **조직의 구성원(직원)**으로 정의하고,  
AI 에이전트들이 역할을 나누어 협업하며 하나의 회사를 운영하는 모델을 실험하는 프로젝트입니다.

기존의 AI 활용 방식이 단일 모델 기반의 자동화에 머물렀다면,  
본 프로젝트는 **AI 에이전트를 인간 전문가 한 명에 대응하는 단위로 정의**하고  
이들이 실제 회사 조직처럼 협업하도록 설계합니다.

본 프로젝트의 핵심 가설은 다음과 같습니다.

> 하나의 AI 에이전트가 인간 전문가 한 명을 대체할 수 있다면,  
> **CEO(사용자)와 HR 에이전트만으로도 하나의 회사를 운영할 수 있지 않을까?**

---

## 2. 개발 배경 및 문제의식

생성형 AI 기술의 발전과 함께 **솔로프레너십(1인 기업)**이 빠르게 확산되고 있습니다.  
하지만 현재의 AI 활용은 대부분 **단발성 작업 수행**에 그치며,  
복합적인 문제 해결이나 장기적인 프로젝트 운영에는 한계가 존재합니다.

본 프로젝트는 다음 질문에서 출발합니다.

> **AI는 인간의 개별 업무를 넘어서, 조직 단위로 문제를 해결할 수 있는가?**

이를 위해 본 프로젝트는  
- 역할과 책임이 명확한 AI 에이전트
- 에이전트 간 협업과 의사결정 구조
- 통제 가능한 확장 방식  

을 갖춘 **AI 회사 모델**을 설계하고 검증합니다.

---

## 3. AI 회사 모델 (ai_company)

### 핵심 역할

- **CEO (사용자)**
  - 회사 목표 설정
  - 백로그 선택
  - 최종 의사결정 및 승인

- **HR 에이전트**
  - 목표 달성에 필요한 역할 분석
  - 역할별 프롬프트를 구성하여 신규 에이전트 생성
  - 에이전트 수와 역할을 통제

- **전문 에이전트**
  - 개발, 기획, QA, 디자인 등 전문 역할 수행
  - 업무 중 막히는 경우 HR 에이전트에게 추가 고용 요청

---

### 기본 동작 흐름

1. CEO가 회사의 목표를 정의
2. HR 에이전트가 필요한 전문 에이전트를 고용
3. 에이전트들이 독립적으로 업무 수행 및 협업
4. 필요 시 새로운 에이전트 확장
5. 반복적인 피드백 루프를 통해 목표 달성

---

## 4. 개발 철학

### 4.1 Spec-Driven Development (SDD)

본 프로젝트는 **SDD(Spec-Driven Development)** 방식을 엄격히 따릅니다.

코드는 다음이 준비되기 전까지 작성되지 않습니다.

- 요구사항 명세
- 유저 스토리
- 기술 스펙
- 테스트 케이스

모든 AI 에이전트는 **명확한 스펙을 기반으로 동작**합니다.

---

### 4.2 Claude Code 기반 개발

본 프로젝트는 **Claude Code**를 핵심 개발 도구로 활용합니다.

- Sub-agents를 통한 역할 분리
- Skills를 통한 반복 작업 추상화
- Plugins를 통한 외부 도구 연동

Claude Code는 단순 보조 도구가 아니라,  
**AI 에이전트 조직을 실행하는 개발 환경**으로 사용됩니다.

---

## 5. SDD 기반 에이전트 개발 파이프라인

실제 제품 조직과 유사한 개발 흐름을 AI 에이전트로 구현합니다.

### 전체 워크플로우

1. **PO (사용자)**
   - 백로그 선택

2. **User Storyteller 에이전트**
   - 백로그를 유저 스토리로 분해

3. **Wireframer 에이전트**
   - 유저 스토리 기반 와이어프레임 작성

4. **PO 검토**
   - 유저 스토리 및 와이어프레임 승인

5. **Tech Lead(TL) 에이전트**
   - 제품 설계 및 기술 스펙 작성
   - 유저 시나리오 포함

6. **QA-TC 에이전트**
   - 테스트 케이스 작성

7. **병렬 작업**
   - PD 에이전트: Figma 디자인 진행
   - BE-Test 에이전트: 테스트 코드 작성
   - BE-Spec 에이전트: OpenAPI Spec 작성

8. **구현 단계**
   - BE-Main 에이전트: API 개발
   - FE-Main 에이전트: 퍼블리싱 및 API 연동

9. **E2E 테스트**
   - QA-E2E 에이전트: Playwright 테스트 코드 작성
   - 테스트 실행

10. **피드백 루프**
    - 버그 리스트 공유
    - FE/BE 개선
    - 반복

11. **최종 승인**
    - PO 최종 컨펌
