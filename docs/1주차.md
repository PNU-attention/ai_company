## 🛠 오늘 진행한 활동

### 1. SDD 개념 학습
- **SDD(Story-Driven Development)** 방법론의 개념과 전체 흐름을 학습함  
- 기능 중심이 아닌 **유저 스토리 중심으로 설계 → 개발 → 테스트**가 이어지는 구조임을 이해함  
- 기존 백엔드 중심 개발 방식과의 차이점을 정리함

### 2. Claude Code 세부 기능 탐색
- Claude Code에서 제공하는 주요 구성 요소 학습
  - **Sub-agents**: 역할 분리 및 병렬 사고를 위한 에이전트 구조
  - **Skills**: 반복 가능한 작업을 모듈화한 실행 단위
  - **Plugins**: 외부 도구 및 시스템과의 연동 방식
- 각 기능이 SDD 프로세스에서 어떤 역할을 할 수 있는지 정리함

### 3. Skill Creator를 활용한 커스텀 Skill 정의 실습
- `skill-creator` skill을 사용하여 커스텀 skill을 직접 정의해봄
- Skill의 목적, 입력값, 출력값, 사용 시나리오를 명확히 설계하는 방식에 익숙해짐
- 향후 반복적으로 사용할 수 있는 내부 전용 skill 설계 가능성을 확인함

### 4. README.md 작성 및 개발 방향성 공유
- 현재 프로젝트의 목적과 구조를 정리한 `README.md` 초안 작성
- Claude Code + SDD 기반 개발 프로세스를 어떻게 적용할지 방향성 공유
- 에이전트 분업 구조 및 향후 구현 순서에 대해 정리함

---

## 📌 오늘의 정리
- SDD는 단순한 개발 기법이 아니라 **사고 방식과 협업 구조를 바꾸는 방법론**임을 인식함  
- Claude Code의 sub-agents / skills 구조는 SDD와 매우 높은 궁합을 가짐  
- 초반에 설계와 역할 정의를 명확히 하는 것이 이후 개발 속도와 품질에 큰 영향을 줄 것으로 판단됨
