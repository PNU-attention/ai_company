# 프로젝트: AI Company

## 개요
- **요청**: AI 에이전트로 구성된 가상의 회사(AI Company)를 운영하는 웹 애플리케이션
- **분석일**: 2026-01-13
- **총 백로그 수**: 9개
- **MVP 범위**: 백로그 1-7

## 서비스 맥락 정의 (모든 백로그에 적용)

- **서비스 유형**: AI Company - AI 에이전트 기반 가상 회사 운영 플랫폼
- **도메인 키워드**: AI 에이전트, 가상 회사, CEO, HR Agent, RM Agent, Expert Agent, 목표 관리, 프로젝트 분해, 작업 분배, 협업 흐름
- **타겟 사용자**: CEO (솔로프레너, 1인 기업가) - 회사 목표를 설정하고 AI 에이전트 조직을 통해 업무를 수행하고자 하는 사용자
- **핵심 가치**:
  - "사람 대신 AI 에이전트로 회사를 운영한다"
  - 목표 → 프로젝트 → 작업(Task) → 실행 → 보고까지의 엔드투엔드 흐름 가시화
  - 프로세스와 역할 흐름이 명확히 보이는 UI
- **구현 제약**:
  - 실제 결과물 생성보다는 프로세스와 역할 흐름 시각화가 중요
  - Human-in-the-loop 구조 유지 (CEO의 의사결정 포인트 필수)
  - SDD(Spec-Driven Development) 철학 반영

> 구현 주의사항: 각 백로그를 구현할 때 반드시 위 서비스 맥락을 참조하세요.
> 예를 들어 "프로젝트 관리" 기능은 "AI 에이전트 기반 가상 회사"의 맥락에서 구현해야 합니다.

## 기술 스택 (권장)

- **Frontend**: Next.js 15+ (App Router, TypeScript, Tailwind CSS)
- **Backend**: FastAPI (Python 3.11+)
- **Database**: PostgreSQL 16+
- **AI Integration**: Claude API (Anthropic)
- **실시간 통신**: Server-Sent Events (SSE) 또는 WebSocket
- **상태 관리**: Zustand
- **기타**:
  - Docker (컨테이너화)
  - pnpm (패키지 관리)
  - pytest (백엔드 테스트)
  - Playwright (E2E 테스트)

## 백로그 목록

### MVP (Phase 1)

#### BL-001: AI Company 프로젝트 초기 설정
- **우선순위**: P0 (Must Have)
- **의존성**: 없음
- **서비스 맥락**: AI 에이전트 기반 가상 회사 운영 플랫폼의 기반을 구축합니다. 모든 후속 기능이 AI Company 도메인에 맞게 동작하도록 합니다.
- **설명**:
  - Next.js 15+ 프론트엔드 프로젝트 생성 (App Router)
  - FastAPI 백엔드 프로젝트 생성
  - PostgreSQL 데이터베이스 설정
  - Docker Compose 개발 환경 구성
  - 기본 프로젝트 구조 및 린팅 설정
- **구현 시 주의**: 일반적인 웹앱이 아닌 AI 에이전트 기반 회사 운영 플랫폼에 특화된 구조로 설정
- **예상 산출물**:
  - `frontend/` - Next.js 프로젝트
  - `backend/` - FastAPI 프로젝트
  - `docker-compose.yml` - 개발 환경
  - ESLint, Prettier, pytest 설정
- **사용 예시**: "CEO가 AI Company 플랫폼에 처음 접속하여 대시보드를 볼 수 있는 기반 구축"

#### BL-002: AI Company 회사 목표 관리
- **우선순위**: P0 (Must Have)
- **의존성**: BL-001
- **서비스 맥락**: CEO가 AI Company에서 회사의 목표를 설정하고 관리하는 핵심 진입점입니다. 모든 에이전트 활동의 시작점이 됩니다.
- **설명**:
  - 회사 목표 입력 UI (목표 제목, 설명, 기한)
  - 목표 목록 조회/수정/삭제
  - 목표 상태 관리 (진행중/완료/보류)
  - 목표 API 엔드포인트 (CRUD)
- **구현 시 주의**:
  - 목표는 "프로젝트"보다 상위 개념으로, HR/RM 에이전트가 분석하는 대상
  - CEO의 의사결정 포인트로서 중요한 UI 설계 필요
- **예상 산출물**:
  - 목표 입력 페이지
  - 목표 목록 대시보드
  - 목표 API (`/api/goals`)
  - Goal 데이터 모델
- **사용 예시**: "CEO가 'AI 기반 고객 서비스 자동화 구축'이라는 회사 목표를 입력하면, 이 목표가 HR 에이전트에게 전달되어 필요한 전문가 에이전트 생성의 기준이 됨"

#### BL-003: AI Company HR 에이전트 - 전문가 에이전트 생성
- **우선순위**: P0 (Must Have)
- **의존성**: BL-001, BL-002
- **서비스 맥락**: AI Company의 핵심 에이전트. 회사 목표를 분석하여 필요한 전문가 에이전트를 동적으로 생성합니다.
- **설명**:
  - HR 에이전트 코어 로직 (Claude API 연동)
  - 목표 분석 → 필요 역할 도출
  - 역할별 프롬프트 생성 및 전문가 에이전트 생성
  - 에이전트 목록 관리 UI
  - 에이전트 생성 과정 시각화
- **구현 시 주의**:
  - HR 에이전트는 "고용자" 역할로, 전문가 에이전트 수와 역할을 통제
  - 에이전트 생성 과정이 UI에 명확히 보여야 함
  - ❌ 일반적인 챗봇이 아닌, ✅ 조직 구성원 생성 시스템
- **예상 산출물**:
  - HR Agent 서비스 (`backend/services/hr_agent.py`)
  - 에이전트 관리 페이지
  - 에이전트 생성 API (`/api/agents`)
  - Agent 데이터 모델
- **사용 예시**: "CEO가 '웹 서비스 개발' 목표를 설정하면, HR 에이전트가 분석하여 'Backend Developer', 'Frontend Developer', 'QA Engineer' 전문가 에이전트를 생성하고, CEO에게 생성된 조직도를 보여줌"

#### BL-004: AI Company RM 에이전트 - 프로젝트 분해 및 계획
- **우선순위**: P0 (Must Have)
- **의존성**: BL-001, BL-002, BL-003
- **서비스 맥락**: 회사 목표를 실행 가능한 프로젝트와 작업(Task)으로 분해하고, 실행 계획을 수립하는 핵심 에이전트입니다.
- **설명**:
  - RM 에이전트 코어 로직 (Claude API 연동)
  - 목표 → 프로젝트 분해
  - 프로젝트 → 작업(Task) 분해
  - 작업 분배 (순차/병렬 정의)
  - 실행 계획 시각화 (Gantt 또는 플로우 다이어그램)
- **구현 시 주의**:
  - 작업 간 의존성 분석이 핵심
  - 순차/병렬 실행 계획이 명확히 보여야 함
  - CEO 승인 포인트 포함
- **예상 산출물**:
  - RM Agent 서비스 (`backend/services/rm_agent.py`)
  - 프로젝트/작업 관리 UI
  - 실행 계획 시각화 컴포넌트
  - Project, Task 데이터 모델
- **사용 예시**: "RM 에이전트가 '웹 서비스 개발' 목표를 '인증 시스템 구축', 'API 개발', 'UI 개발' 프로젝트로 분해하고, 각 프로젝트의 세부 작업과 담당 에이전트, 실행 순서를 계획표로 보여줌"

#### BL-005: AI Company Expert 에이전트 - 작업 수행
- **우선순위**: P0 (Must Have)
- **의존성**: BL-001, BL-003, BL-004
- **서비스 맥락**: HR 에이전트가 생성한 전문가 에이전트가 RM 에이전트가 할당한 작업을 수행하는 실행 단위입니다.
- **설명**:
  - Expert 에이전트 기본 프레임워크
  - 작업 수행 로직 (Claude API 연동)
  - 작업 진행 상태 업데이트
  - 전문성 초과 시 RM에게 요청 기능
  - 작업 결과물 생성 및 저장
- **구현 시 주의**:
  - 각 Expert 에이전트는 특정 역할(개발자, QA 등)에 특화
  - 작업 수행 과정이 실시간으로 UI에 표시되어야 함
  - 에이전트 간 협업(요청/응답) 흐름 시각화
- **예상 산출물**:
  - Expert Agent 베이스 클래스
  - 역할별 Expert Agent 구현 (Developer, QA, Designer 등)
  - 작업 실행 API (`/api/tasks/{id}/execute`)
  - 작업 결과 저장소
- **사용 예시**: "Backend Developer 에이전트가 '사용자 인증 API 개발' 작업을 수행하고, 진행 상황(분석 중 → 설계 중 → 구현 중 → 완료)이 실시간으로 대시보드에 표시됨. 작업 중 DB 설계 전문성이 필요하면 RM에게 DBA 에이전트 추가 요청"

#### BL-006: AI Company 워크플로우 대시보드
- **우선순위**: P0 (Must Have)
- **의존성**: BL-001, BL-002, BL-003, BL-004, BL-005
- **서비스 맥락**: CEO가 AI Company의 전체 운영 현황을 한눈에 파악할 수 있는 중앙 대시보드입니다. 목표 → 프로젝트 → 작업 → 실행 → 보고의 엔드투엔드 흐름을 시각화합니다.
- **설명**:
  - 회사 운영 현황 개요 (활성 목표, 진행중 프로젝트, 에이전트 상태)
  - 에이전트 조직도 시각화
  - 작업 흐름 실시간 모니터링
  - 프로젝트/작업 진행률 표시
  - 에이전트 간 협업 흐름 시각화
- **구현 시 주의**:
  - 프로세스와 역할 흐름이 명확히 보이는 UI가 핵심
  - 실시간 업데이트 (SSE 또는 WebSocket)
  - CEO가 직관적으로 현황 파악 가능해야 함
- **예상 산출물**:
  - 메인 대시보드 페이지
  - 에이전트 조직도 컴포넌트
  - 작업 흐름 다이어그램 컴포넌트
  - 실시간 상태 업데이트 시스템
- **사용 예시**: "CEO가 대시보드에서 현재 '웹 서비스 개발' 목표가 60% 진행 중이고, 5개의 에이전트가 활동 중이며, 'API 개발' 프로젝트에서 Backend Developer와 QA가 협업 중인 것을 플로우 차트로 확인"

#### BL-007: AI Company 실행 결과 보고 시스템
- **우선순위**: P0 (Must Have)
- **의존성**: BL-001, BL-004, BL-005, BL-006
- **서비스 맥락**: RM 에이전트가 작업 완료 후 CEO에게 최종 결과를 보고하는 시스템입니다. AI Company 운영 사이클의 마무리 단계입니다.
- **설명**:
  - 프로젝트/목표 완료 보고서 자동 생성
  - 에이전트별 수행 내역 요약
  - 성과 지표 (완료율, 소요 시간 등)
  - 보고서 조회 및 내보내기
  - CEO 피드백 입력
- **구현 시 주의**:
  - 보고서는 CEO가 의사결정하기 좋은 형태로 구성
  - 에이전트별 기여도가 명확히 보여야 함
  - 반복적인 피드백 루프를 위한 구조
- **예상 산출물**:
  - 보고서 생성 서비스
  - 보고서 조회 페이지
  - 보고서 API (`/api/reports`)
  - Report 데이터 모델
- **사용 예시**: "'웹 서비스 개발' 목표 완료 후, RM 에이전트가 자동으로 보고서를 생성하여 CEO에게 '총 15개 작업 완료, 5명의 에이전트 참여, 예상 대비 2일 단축' 등의 성과를 보고"

## 참고사항

### 프로젝트 특이사항
- **SDD(Spec-Driven Development)**: README에 명시된 대로, 모든 기능은 스펙이 먼저 정의된 후 구현
- **Human-in-the-loop**: CEO의 승인/피드백 포인트가 워크플로우에 필수로 포함
- **시각화 중심**: 실제 결과물 생성보다 프로세스 흐름 가시화가 핵심
- **24시간 운영**: 장기적으로 지속적인 운영을 목표로 하는 구조 설계 필요

### 기술적 고려사항
- **Claude API 비용 관리**: 에이전트 호출 시 토큰 사용량 추적 필수
- **실시간 업데이트**: SSE 또는 WebSocket을 통한 대시보드 실시간 갱신
- **확장성**: 에이전트 타입 추가가 용이한 플러그인 구조 권장
- **로깅**: 모든 에이전트 활동 로그 저장 (디버깅 및 분석용)

### 향후 확장 고려
- 에이전트 메모리 및 학습 (컨텍스트 유지)
- 외부 도구 연동 (GitHub, Slack 등)
- 멀티 테넌트 (여러 회사 운영)
- AI 회사 운영 SaaS로 확장
